1.常见的页面优化技术

课程中列出了如下几种页面优化方案：

  • 页面缓存、URL 缓存、对象缓存

    页面缓存就是将页面代码加入缓存，首先从缓存中读取，娶不到就手动渲染。URL 缓存就是具体一点的，如果说商品列表用页面缓存，那么商品详情就用 URL 缓存，因为每个商品的详情页不一样，其实 URL 缓存就是页面缓存。对象缓存，是粒度最小的一种。

  • 页面静态化

    页面都是用 HTML 写的，这些 HTML 代码全部缓存在浏览器中，请求通过 js 或者 ajax 来发起。

下面我们来看如何实现。

优化时，我们先从缓存中获取页面，如果取不到，那就使用 ThymeleafViewResolver 的 getTemplateEngine().process 和我们获取到的数据渲染页面，同时将页面加入缓存。

注意：

在代码中我们注入了 thymeleafViewResolver，这个是框架中已经有的，我们只要直接注入，再传入页面以及一个 WebContext 对象，这是在 springboot 2.4.4 中所使用的。

此外，@RequestMapping(value = “to_list”, produces = “text/html”) 注解中用到的 produces = “text/html” 标注了返回的样式。而且页面缓存的时间也不宜过长，不然时效性会很差，所以不要忘记设置一个过期时间。

2.3.对象缓存

上面的页面缓存适合时间比较短，变化不大的场景。而相比于页面缓存，对象缓存的粒度更小，他只缓存一个对象。这个好处就是：当需要用户数据的时候，可以直接从缓存中拿。如下：

    public SeckillUser selectById(long id){
        // 取缓存
        SeckillUser user = redisService.get(SeckillUserKey.selectById, "" + id, SeckillUser.class);
        if(user != null){
            return user;
        }
        // 取数据库
        user = seckillUserDao.selectById(id);
        if(user != null){
            redisService.set(SeckillUserKey.selectById, "" + id, user);
        }
        return user;
    }

通过上面的操作，缓存中已经有了 token——user、id——user 这样的数据了。

但是如果用户更新了信息，比如修改密码这样的场景，此时，为了保证数据库和缓存的一致性，数据库中密码修改后，缓存中也需要进行修改！但是此时需要注意，token——user 需要修改用户的密码，而 id——user 直接删除就好了，token 是需要用到的。

但是一般的网站在修改密码之后，都会要求重新登录，所以两个都直接删除也是可以的！

    public boolean updatePassword(String token, long id, String formPass) {
        //取user
        SeckillUser user = selectById(id);
        if(user == null) {
            throw new GlobalException(CodeMsg.MOBILE_NOT_EXIST);
        }
        //更新数据库
        SeckillUser toBeUpdate = new SeckillUser();
        toBeUpdate.setId(id);
        toBeUpdate.setPassword(MD5Util.formPassToDBPass(formPass, user.getSalt()));
        seckillUserDao.update(toBeUpdate);
        //处理缓存
        redisService.delete(SeckillUserKey.selectById, "" + id);
        user.setPassword(toBeUpdate.getPassword());
        redisService.set(SeckillUserKey.token, token, user);
        return true;
    }

3.页面静态化

页面静态化通常是将页面代码缓存到浏览器中，不需要与服务器进行交互就可以浏览到网页。其中动态变化的数据靠 ajax 和 js 来请求。
